Challenge 1: Rating pop-up

I found a website that hosts a lot of different kind of challenges. Most of these have something to do with either CSS, JavaScript or HTML or a combination of these. You can upload your solution and get feedback on this from other people. These challenges can help me grow as an individual and grow more confident in my ability to code without the use of tutorials. ()

The first one I took on was a rating pop-up window where you can click on a certain number, this number then changes color and you can submit this, from where it goes to a different state to thank the user for submitting their review. In this state, it has to show the correct number that was clicked too.

The main things that I learned from this are:

  • The different ways in which to use a click-function. The way I did it at the moment works, but isn't the most professional. At the moment I basically said that when a button is clicked, a certain function has to go off. The more professional way to handle this would be with an event listener.
  • Way of thinking. Before this, I've only done tutorials or very easy JS functions during lectures. For example, last semester I created a snake game. And where I was very proud of what I created here and I learned a lot, it was still following a tutorial. By not doing this, I learned to think in a way that helps me come up with certain ideas on how to solve the problem I am having. I wasn't always able to solve these things without going on google to look for different ways to solve it, but in the end I did do all the thinking myself and I am very proud of that.

I really enjoyed working on this challenge and I learned a lot from it. I think that by doing these challenges and working my way up to the more difficult ones, I can grow as not just a person. But also as a front-end developer.

I also learned more about Github and the function they have in which you can host the webpage with them for free. This is something that the website uses so other people can review your website, but this was also a new experience for me and something I found very interesting and a concept that I think is really nice.

On top of that, I also got in touch with readme's for the first time, and learned more about what they are, what they're used for and why they're important. During this challenge I also worked a bit on commenting my code and learning more about how to do this in such a way that it helps people understand your code. ( )
